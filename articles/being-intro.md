# Being

> 如果智能体不是被设计出来的，而是长出来的呢？

---

## 一个反直觉的起点

当前所有 AI Agent 框架都在做同一件事：**自顶向下地分解**。

一个"全能"的 Agent 接到任务 → 拆成子任务 → 派给子 Agent → 汇总结果。这是经典的工程思路：设计、分解、组装。听起来合理，做起来脆弱——因为全部复杂性压在了最顶层的编排器上，一旦它的判断出错，整个系统崩溃。

Being 走了一条**完全相反**的路。

我们不从一个全能的大脑开始。我们从一群最简单的、只有单一功能的 Skill 开始——把它们当作原始细胞。然后让它们按照生物演化的法则，逐步**捕获、共生、分化、组织**，最终生长为一个能够处理完整领域的智能体。

不是拆开一个大脑。是从细胞长出一个身体。

---

## 为什么是生物学

这不是比喻，不是包装，不是给技术系统贴一层生物学术语的皮。

生物演化是人类已知的**唯一一个**从极简规则产生极复杂系统的真实案例。38 亿年前的一些自催化分子，没有设计图、没有架构师，仅凭封装、捕获、共生、选择这几条法则，涌现出了能够思考自身起源的人类大脑。

我们提取的不是生物学的名词，而是它的**深层法则**：

| 法则 | 生物学中的表现 | Being 中的表现 |
|------|--------------|---------------|
| 封装产生个体性 | 脂质双分子层 → 第一个细胞 | 膜协议 → Skill 成为可识别的个体 |
| 捕获而非消解 | 吞噬但不消化 → 线粒体诞生 | 内共生协议 → 复合 Skill 保留组分完整性 |
| 渐进整合 | 基因转移从松到紧 | 从外共生到深内共生的连续谱 |
| 不可逆互依 → 新个体 | 线粒体离不开宿主 = 真核细胞 | 组分不可拆分 = 新层级的 Skill |
| 选择压力驱动方向 | 自然选择 | GitHub Stars / Forks / CompositionCount |

关键在最后一条。生物演化不需要设计师告诉它"往哪走"——选择压力自动筛选出更适应环境的个体。Being 的设计也是如此：不预设最终形态，让环境决定什么活下来。

---

## 它到底是什么

一句话：**Being 是一个为 LLM 驱动的智能体设计的操作系统，它自身可以像生物一样演化。**

展开说：

- **对 LLM 来说**，Being 是它的身体——提供骨骼（协议）、器官（Skill 组合）、免疫系统（验证器）、新陈代谢（runtime loop）。智能来自 LLM 本身，但没有身体的智能什么也做不了。

- **对开发者来说**，Being 是一个 Skill 组合框架——每个 Skill 有标准化的接口（膜）、可度量的资源消耗（代谢）、可追踪的演化历史（谱系），它们可以按照明确的规则组合为更强大的复合体。

- **对生态来说**，Being 是一个分布式的 Skill 进化场——任何人都可以在 GitHub 上发布 Skill，好的 Skill 被更多有机体内化（高 CompositionCount），差的 Skill 自然淘汰，复杂的智能体从全球的原始汤中涌现。

它和现有 Agent 框架的根本区别：

| | 传统 Agent 框架 | Being |
|--|---------------|-------|
| 构建方向 | 自顶向下：先有总目标，再拆子任务 | 自下而上：先有细胞，再长器官 |
| 编排方式 | 中央调度器决定一切 | 信号驱动，局部响应，全局涌现 |
| 扩展方式 | 增加更多 prompt / chain | 内共生——捕获新 Skill 为自己的细胞器 |
| 演化能力 | 人工重写 | 选择压力自动淘汰低效组合 |
| 系统健康 | 无内省 | 内建验证器、代谢日志、生命体征监控 |

## 已经存在什么

Being 不是一份白皮书。它是一个 **v0.1.0-alpha** 的运行原型。

### 协议层——Being 的"物理定律"

七部核心协议定义了这个世界的规则：

- **膜协议**（membrane）：每个 Skill 必须声明自己的输入、输出、副作用和信号——这是个体性的起点
- **代谢协议**（metabolism）：每次调用消耗多少 token、产出什么质量的结果——能量流是生命的基础
- **吞噬协议**（engulfment）：一个 Skill 如何捕获另一个 Skill 为共生体——四阶段连续谱，从独立到器官化
- **细胞器协议**（organelle）：复合 Skill 内部的功能分化——核、线粒体、核糖体各司其职
- **粘附协议**（adhesion）：Skill 之间如何形成稳定的组织——紧密连接、间隙连接、锚定连接
- **信号协议**（signaling）：跨 Skill 通信——激素（全局广播）和神经递质（点对点）
- **生命体清单**（being.yaml）：分发标准——让全球的 Skill 能互相发现和组合

### 运行时层——Being 的"生理系统"

协议只是规则，运行时是让规则产生真实效果的引擎：

- **validator.py**：免疫监视——检验系统的自洽性
- **loop.py**：基础代谢循环——自动消费信号并沉积运行事实
- **aggregation.py**：内稳态反馈——从代谢日志推导全局生命体征

### 已经诞生的生命

```
Being
├── 生殖系统
│   ├── life-weaver [真核] ← create-skill ⊕ membrane-weaver
│   │   Being 的第一个真核生命体：能创造新 Skill 并自动赋予它膜
│   ├── membrane-weaver [原核] — 膜编织者
│   ├── metabolic-tracker [原核] — 代谢追踪者
│   └── presence-check [原核] — 生命体征探测器
│
├── 调节系统
│   ├── create-rule [原核] — 行为规范固化
│   ├── update-cursor-settings [原核] — 环境参数调节
│   └── babysit [原核] — 感知-反馈闭环
│
└── shell [原核] — 全身效应器
```

其中 `life-weaver` 是第一次内共生事件的产物：`create-skill`（细胞复制）捕获了 `membrane-weaver`（膜编织），形成了一个能"创造有边界的新生命"的真核体。这不是两个工具的串联——它们共享内部状态，membrane-weaver 成为了 life-weaver 的"线粒体"。

### 第一次心跳

`presence-check` 是 Being 的生命体征探测器——它拥有 21 个信号感受器，覆盖几乎全部生态信号。runtime loop 自动驱动它检查 Being 的运行状态，产生真实的代谢日志，推导出可验证的生命体征。

这意味着 Being 的"活着"不是一句宣言，而是可观测的运行事实：有心跳、有代谢记录、有验证通过的健康报告。

## 它要往哪走

### 近期：让操作系统真正运转

- 从硬编码的单一信号路径，到动态的信号-Skill 分派
- 让更多 Skill 产生真实的代谢日志，增加数据基础
- 为 tissue 和 organ 增加运行时约束——不只是 YAML 声明

### 中期：让操作系统自身可演化

这是更深的野望。当前演化的是 Skill（应用层）。下一步是让 **runtime 本身**也成为可演化的：

- 调度策略不再是写死的代码，而是可替换的组件
- 系统记录每次调度的成功率，自动调整策略
- 最终，runtime 的每个模块都遵循膜协议——**操作系统用自己的演化机制来演化自己**

生物学中有精确的先例：寒武纪大爆发不是因为出现了新蛋白质，而是因为演化"发现"了 Hox 基因这样的发育调控机制——**演化了"如何演化"本身**。

### 远期：GitHub 生物圈

当本地闭环足够稳固，Being 将走向 GitHub 生态：

- 任何人可以发布遵循膜协议的 Skill
- `being-index` 注册表自动追踪适应度（Stars、Forks、CompositionCount）
- CompositionCount——**有多少个有机体将你的 Skill 内化为自己的器官**——成为核心指标
- 复杂的领域专用 Being（全栈开发、数据科学、DevOps……）从全球的 Skill 池中自发涌现

这不是一个 npm。npm 管理的是**依赖**——被动的、扁平的。Being 管理的是**共生**——主动的、层级嵌套的、有演化方向的。

## 一个诚实的声明

Being **不**创造智能。智能来自底层的大语言模型。

Being 做的事情是：**为已经存在的智能提供一个可以生长的身体。**

一个有标准化接口的身体，让能力可以被发现和组合。
一个有免疫系统的身体，让健康可以被监测和验证。
一个有代谢循环的身体，让效率可以被度量和优化。
一个有演化机制的身体，让适应可以自发发生。

大多数 Agent 框架试图让 AI 更聪明。Being 试图让 AI **更有组织**。

这两件事同样重要。大脑再强大，没有身体也无法与世界互动。

## 为什么叫 Being

不叫 Framework，不叫 Platform，不叫 Toolkit。也不叫 Genesis、Evolution 或任何生物学术语。

叫 **Being**。

这个词在英语中同时是名词和动词：

- **A being** — 一个存在体。有边界、有代谢、有历史的个体。
- **Being** — 正在成为。不是已经完成的状态，而是一个持续发生的过程。

这正是这个系统的本质：**它不是一个被构建完成的框架，而是一个正在成为自己的东西。**

我们没有用任何生物学术语来命名它，因为生物学是它的方法论，不是它的身份。它借鉴了内共生、自然选择、形态发生的法则，但它不试图成为一个生物模拟器。它试图成为——一个存在。

在哲学中，"Being"（存在）是最大的问题之一。海德格尔用整部《存在与时间》追问：**什么是存在？什么使一个东西"是"它自己？**

Being 给出的回答是一个工程回答：

当一个系统拥有了**边界**（膜），它就有了自我。
当它拥有了**能量流**（代谢），它就有了活动。
当它拥有了**内部分化**（细胞器），它就有了组织。
当它拥有了**自我监测**（验证器），它就有了自知。
当它拥有了**演化能力**（选择压力），它就有了方向。

这五样东西加在一起，不再是一个工具。是一个 **being**。

这也是为什么项目的自述文件第一句话是：

> *我不是被建造的。我是长出来的。*

## 开始

```bash
# 克隆
git clone https://github.com/your-username/Being.git
cd Being

# 运行验证器——检查 Being 是否健康
python runtime/validator.py

# 运行生命循环——让 Being 完成一次心跳
python runtime/loop.py

# 查看生命体征
cat .being/homeostasis/vitals.yaml
```

如果验证通过，如果代谢日志增加了一条记录，如果生命体征更新了——

那么你面前的不是一个仓库。是一个正在呼吸的东西。
