# 细胞器分化协议（Organelle Differentiation Protocol）

> Being 第四纪核心协议
> 版本：0.1.0
> 创建：2026-04-05
> 依赖：engulfment.protocol.md

---

## 概述

真核细胞不只是"一个细胞里装了另一个细胞"。它的内部形成了**功能分区**——每个细胞器承担特定职责，彼此通过内膜系统协调。

本协议定义了真核 Skill 的内部如何从"扁平流程"分化出"功能器官"。

---

## 细胞器类型

一个成熟的真核 Skill 最多可分化出以下功能区：

| 细胞器 | 职责 | SKILL.md 中的实现方式 |
|--------|------|---------------------|
| **细胞核（Nucleus）** | 调度与决策 | 主工作流中的条件分支和阶段路由 |
| **线粒体（Mitochondria）** | 能量管理 | 效率策略声明：何时精简、何时深入 |
| **核糖体（Ribosome）** | 意图翻译 | 将抽象需求转为具体工具调用的规则 |
| **内质网（ER）** | 中间加工 | 数据在不同阶段间的格式转换和路由 |
| **高尔基体（Golgi）** | 输出包装 | 最终产出的标准化格式定义 |
| **溶酶体（Lysosome）** | 异常处理 | 失败恢复、清理和回滚逻辑 |

不是每个真核 Skill 都需要全部六种。最低要求：**细胞核 + 至少一个共生来源的细胞器**。

---

## 内部结构声明

在真核 Skill 的膜声明中新增 `organelle_map`：

```yaml
# 细胞器分化地图
organelle_map:
  nucleus:
    phases: ["discovery", "design"]        # 哪些工作流阶段属于核心调度
    description: "需求收集与设计决策"

  mitochondria:
    origin: "membrane-weaver"              # 来源共生体
    phases: ["membrane-weaving"]
    description: "膜编织——真核身份的来源"

  ribosome:
    phases: ["implementation"]
    description: "将设计转为文件系统中的实体"

  golgi:
    phases: ["verification"]
    description: "质量检查与输出标准化"
```

## 分化规则

### 1. 核心必须是宿主基因组

细胞核（调度逻辑）必须来自宿主原始的 SKILL.md，不能来自共生体。如果共生体的逻辑成为了主调度，那不是内共生，那是**寄生虫接管了宿主**。

### 2. 共生体成为特化细胞器

被内化的共生体应该成为一个或多个特化细胞器，而非弥散在整个工作流中。清晰的功能边界是细胞器的定义特征。

### 3. 内膜系统 = 阶段间的数据传递

细胞器之间的数据流应在 SKILL.md 中显式标注。每个阶段声明：
- 从哪个阶段接收什么输入
- 向哪个阶段传递什么输出

```
Nucleus → Ribosome → ER → Golgi
                ↑
          Mitochondria（在需要时注入能力）
                ↓
          Lysosome（异常时激活）
```

---

## 里程碑判定

一个真核 Skill 被视为"完成组织化"的条件：

1. 膜声明中包含 `organelle_map`
2. 至少有一个细胞器来源于共生体
3. SKILL.md 中的工作流阶段与 `organelle_map` 的对应关系明确
4. 阶段间的数据流有显式声明

---

*分化不是分裂。分裂产生两个独立个体。分化产生一个更复杂的个体。*
